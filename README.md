# 每日自动获取AI简讯agent

## 项目概述

这是一个自动化的AI行业简讯生成系统，能够从多个数据源自动采集AI相关新闻，经过智能处理后生成高质量的每日简报，并支持定时自动运行和GitHub Pages部署。

## 核心功能

- **全自动定时**：内置GitHub Actions定时任务，支持每日8:00和8:30自动触发
- **多数据源支持**：支持RSS源和微信公众号数据源
- **智能内容处理**：自动分类、去重、优先级排序
- **多格式输出**：生成HTML网页、Markdown文档、JSON数据
- **响应式设计**：适配手机和电脑端访问
- **分享功能**：一键分享今日简讯内容
- **历史数据查看**：支持查看历史简讯统计

## 输入输出文件规范

### 固定输入文件
- `source/AI_sources.xlsx` - 数据源配置文件
- `source/resultAI.csv` - 数据源统计文件

### 固定输出文件
- `index.html` - 主网页文件（根目录）
- `latest.md` - 最新简讯Markdown文档
- `source/resultAI.csv` - 数据源获取统计
- `source/resultAI.json` - JSON格式统计文件

## 数据源配置

### 数据源格式
数据源配置文件 `source/AI_sources.xlsx` 包含以下字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| name | 数据源名称 | 阿里云（微信公众号） |
| type | 数据源类型 | rss |
| rss_url | RSS地址 | https://wechatrss.waytomaster.com/api/rss/... |
| home_url | 主页地址 | https://mp.weixin.qq.com/... |
| category | 分类 | 云计算 |
| priority | 优先级 | 10 |
| enabled | 是否启用 | 1 |

### 11个固定分类
1. **大模型**：LLM、GPT、千问、文心一言、通义等
2. **AI Agent**：智能体、Agent平台、自动化等
3. **算力**：GPU、TPU、AI芯片、计算卡、数据中心等
4. **政策合规**：政策、法规、标准、监管、安全等
5. **行业方案**：行业应用、解决方案、落地案例等
6. **云计算**：云服务、云平台、云原生、云安全等
7. **开源**：开源项目、开源工具、开源框架等
8. **商业化**：商业模式、商业应用、市场机会等
9. **安全**：网络安全、数据安全、应用安全等
10. **企业服务**：企业应用、企业软件、企业平台等
11. **技术趋势**：技术发展、技术创新、技术突破等

## 内容处理流程

1. **数据采集**：从配置的数据源获取最新文章
2. **日期筛选**：只保留最近发布的文章
3. **智能分类**：基于关键词匹配到11个固定类别
4. **优先级计算**：根据技术变化、商机、成本等因素排序
5. **内容生成**：生成标题、摘要、云厂商视角点评
6. **格式输出**：生成HTML网页和Markdown文档
7. **统计记录**：记录数据源获取情况和统计信息

## 简讯内容规范

### 标题要求
- 提炼核心内容，避免标题党性质的夸张描述
- 基于原文标题，去掉夸张描述部分

### 摘要要求（200-300字）
- 客观描述，不做分析
- 包含主体 + 时间 + 核心动作 + 相关数据
- 在原文基础上自动提炼关键信息

### 点评要求（2-3句话）
- 从云厂商视角做简要分析
- 分析事件对行业影响、竞争格局、潜在发展趋势
- 避免口号式表达

## 数据源统计管理

### 统计文件格式
`source/resultAI.csv` 包含以下统计信息：

| 字段名 | 说明 |
|--------|------|
| 获取简讯数 | 成功获取的简讯数量 |
| 获取结果 | 成功/失败原因说明 |
| 生成时间 | 数据生成时间戳 |
| 最新简讯发布时间 | 最新简讯的发布时间 |

### 部分数据源处理
当仅获取到部分数据源信息时，统计文件仅呈现成功获取的数据源信息，包括：
- 成功获取简讯的数据源详情
- 获取失败的数据源及失败原因
- 最新简讯发布时间信息

## 故障排除

### 常见问题

1. **数据源获取失败**
   - 检查网络连接
   - 验证RSS地址有效性
   - 查看错误日志

2. **网页显示异常**
   - 清除浏览器缓存
   - 检查GitHub Pages状态
   - 验证HTML文件格式

3. **定时任务未执行**
   - 检查GitHub Actions配置
   - 查看工作流运行历史
   - 验证仓库权限设置
