# AI简讯自动更新定时任务设置说明

## 📋 概述

本系统已配置完成，可以实现每天8:00和8:30自动获取最新AI简讯并上传到GitHub。

## 🛠️ 已创建的脚本文件

### 1. 自动更新脚本 (`auto_update_news.bat`)
- **功能**: 自动生成简讯、提交到Git、推送到GitHub
- **包含**: 环境检查、错误处理、日志记录
- **运行时间**: 约1-2分钟

### 2. 定时任务设置脚本 (`setup_scheduled_task.ps1`)
- **功能**: 自动创建Windows定时任务
- **要求**: 需要管理员权限运行

### 3. 测试脚本 (`test_auto_update.bat`)
- **功能**: 测试环境和功能是否正常
- **用途**: 设置前验证

## ⚙️ 设置步骤

### 第一步：测试环境
1. 双击运行 `test_auto_update.bat`
2. 确保所有测试都通过
3. 如果测试失败，请检查Python和Git安装

### 第二步：设置定时任务（需要管理员权限）

#### 方法一：使用PowerShell脚本（推荐）
1. **以管理员身份运行PowerShell**
   - 在开始菜单搜索"PowerShell"
   - 右键点击"以管理员身份运行"

2. **切换到脚本目录并运行设置脚本**
   ```powershell
   cd "C:\Users\h604658591\Demo2"
   .\setup_scheduled_task.ps1
   ```

#### 方法二：手动设置（备用）
1. 打开"任务计划程序"
   - 按Win+R，输入`taskschd.msc`

2. 创建基本任务
   - 右键"任务计划程序库" → "创建基本任务"
   - 名称: `AI简讯自动更新-08:00`
   - 触发器: 每天，8:00
   - 操作: 启动程序
     - 程序: `cmd.exe`
     - 参数: `/c "C:\Users\h604658591\Demo2\auto_update_news.bat"`
     - 起始于: `C:\Users\h604658591\Demo2`

3. 重复步骤创建第二个任务
   - 名称: `AI简讯自动更新-08:30`
   - 触发器: 每天，8:30

## 🔧 定时任务配置详情

### 任务1: AI简讯自动更新-08:00
- **名称**: `AI简讯自动更新-08:00`
- **描述**: 每天8:00自动获取AI简讯并更新到GitHub
- **触发器**: 每天 08:00
- **操作**: 运行 `auto_update_news.bat`

### 任务2: AI简讯自动更新-08:30
- **名称**: `AI简讯自动更新-08:30`
- **描述**: 每天8:30自动获取AI简讯并更新到GitHub
- **触发器**: 每天 08:30
- **操作**: 运行 `auto_update_news.bat`

## 📊 自动更新流程

每次定时任务运行时，将执行以下步骤：

1. **环境检查** ✅
   - 检查Python和Git是否可用

2. **生成简讯** 🚀
   - 运行 `generate_fixed_news.py`
   - 生成HTML和Markdown文件

3. **Git提交** 📝
   - 添加所有更改到Git
   - 提交并包含时间戳

4. **GitHub推送** 📤
   - 推送到GitHub仓库
   - 更新GitHub Pages

5. **日志记录** 📋
   - 记录运行状态到 `auto_update.log`

## 🔍 监控和管理

### 查看定时任务状态
1. 打开"任务计划程序" (`taskschd.msc`)
2. 导航到: 任务计划程序库 → AI简讯自动更新-08:00/AI简讯自动更新-08:30
3. 查看"上次运行结果"和"下次运行时间"

### 查看运行日志
- **脚本日志**: `auto_update.log`
- **Git提交记录**: GitHub仓库提交历史
- **系统日志**: 任务计划程序中的历史记录

### 手动运行测试
```powershell
# 手动运行定时任务
Start-ScheduledTask -TaskName "AI简讯自动更新-08:00"

# 或直接运行批处理文件
.\auto_update_news.bat
```

## ⚠️ 注意事项

### 系统要求
- Windows 10/11 系统
- Python 3.x 已安装并添加到PATH
- Git 已安装并配置好GitHub凭据
- 稳定的网络连接

### 权限要求
- 设置定时任务需要管理员权限
- 定时任务运行时需要用户已登录

### 网络要求
- 需要能够访问GitHub
- 需要能够访问数据源网站（用于获取简讯）

## 🛠️ 故障排除

### 常见问题

1. **定时任务不运行**
   - 检查任务计划程序中的任务状态
   - 确保系统时间正确
   - 检查用户是否已登录

2. **Git推送失败**
   - 检查网络连接
   - 验证GitHub凭据
   - 检查Git远程仓库配置

3. **Python脚本错误**
   - 检查Python安装
   - 验证脚本路径
   - 查看错误日志
